### 架构设计
```
┌─────────────────────────────────────────────────────┐
│                  客户端 (Web/移动端)                  │
└──────────────────────────┬──────────────────────────┘
                           │ WebSocket (WSS)
┌──────────────────────────▼──────────────────────────┐
│              WebSocket 服务器 (NestJS)               │
│  ├────────────────────────────────────────────────┤  │
│  │  • 连接管理      • 认证授权      • 消息路由      │  │
│  │  • 心跳检测      • 频道订阅      • 广播推送      │  │
│  └────────────────────────────────────────────────┘  │
└─────────────┬────────────────────────────┬──────────┘
              │ Redis Pub/Sub              │ HTTP API
    ┌─────────▼─────────┐        ┌─────────▼─────────┐
    │    Redis 集群     │        │   业务逻辑服务器   │
    │  • 消息队列       │        │  • 任务管理       │
    │  • 会话存储       │        │  • 账号管理       │
    │  • 状态缓存       │        │  • 系统监控       │
    └───────────────────┘        └───────────────────┘
```

## 🚀 部署准备

### 环境要求
- **Node.js**: >= 18.0.0
- **Redis**: >= 6.0.0
- **内存**: 4GB+ (生产环境建议 8GB+)
- **磁盘**: 10GB+ 可用空间

### 部署步骤
1. **环境准备**: 安装 Node.js 和 Redis
2. **配置设置**: 配置环境变量和 Redis
3. **服务部署**: 部署 WebSocket 服务器和推送服务
4. **监控配置**: 设置监控和告警
5. **测试验证**: 运行测试验证系统功能

### 配置文件示例
```bash
# .env 配置文件
NODE_ENV=production
WS_PORT=3002
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=your-strong-password
JWT_SECRET=your-secure-jwt-secret
FRONTEND_URL=https://your-domain.com
```
