# Facebook Auto Bot - 项目启动指南

## 🚀 项目概述

Facebook Auto Bot 是一个多环境账号操作调度平台，专门为管理多个Facebook账号的自动化互动设计。系统通过模拟人类行为，实现账号间的自动对话、内容发布和互动管理。

## 📋 核心功能

- **多账号管理**: 支持10个Facebook账号独立管理
- **自动聊天系统**: 50个预定义对话剧本，自然对话节奏模拟
- **智能调度**: 秒级精度任务调度，并发控制
- **用户友好界面**: PWA优先，支持离线功能
- **数据安全**: 失败记录7天自动清理，端到端加密

## 🏗️ 技术架构

### 后端技术栈
- **框架**: NestJS + TypeScript
- **数据库**: PostgreSQL 16 + Redis 7
- **消息队列**: RabbitMQ
- **对象存储**: MinIO (S3兼容)
- **部署**: Docker + Kubernetes

### 前端技术栈
- **框架**: React 19 + TypeScript
- **构建工具**: Vite
- **UI组件**: Ant Design 5
- **状态管理**: Zustand
- **PWA**: 支持离线功能和推送通知

## 🛠️ 快速开始

### 环境要求
- Docker 24.0+
- Docker Compose 2.20+
- Node.js 20+ (可选，用于本地开发)

### 一键启动开发环境
```bash
# 克隆项目
git clone <repository-url>
cd facebook-auto-bot

# 启动所有服务
./start-dev.sh up

# 查看服务状态
./start-dev.sh status
```

### 分步启动
```bash
# 1. 安装依赖
./start-dev.sh install

# 2. 数据库迁移
./start-dev.sh db migrate

# 3. 种子数据
./start-dev.sh db seed

# 4. 启动后端开发服务器
./start-dev.sh backend dev

# 5. 启动前端开发服务器
./start-dev.sh frontend dev
```

## 🔧 开发命令

### 常用命令
```bash
# 启动开发环境
./start-dev.sh up

# 停止开发环境
./start-dev.sh down

# 重启服务
./start-dev.sh restart

# 查看日志
./start-dev.sh logs backend
./start-dev.sh logs frontend

# 数据库操作
./start-dev.sh db migrate    # 运行迁移
./start-dev.sh db seed       # 种子数据
./start-dev.sh db backup     # 备份数据库
./start-dev.sh db restore    # 恢复数据库

# 清理环境
./start-dev.sh clean
```
