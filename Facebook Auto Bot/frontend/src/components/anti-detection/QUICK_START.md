# 反检测配置管理面板 - 快速启动指南

## 立即开始

### 1. 访问面板
1. 启动前端开发服务器：`npm run dev`
2. 打开浏览器访问：`http://localhost:5173`
3. 登录系统
4. 点击左侧导航栏的"反检测配置"菜单

### 2. 创建第一个配置
1. 点击"创建配置"按钮
2. 选择"预设模板"标签页
3. 点击"桌面 Chrome (Windows)"模板
4. 切换到"自定义配置"标签页
5. 点击"创建配置"按钮

### 3. 测试配置
1. 在配置列表中找到刚创建的配置
2. 点击"测试"按钮（播放图标）
3. 点击"开始测试"按钮
4. 查看测试结果

## 常用模板

### 推荐模板
1. **桌面 Chrome (Windows)** - 最常用，成功率95%
2. **移动 Chrome (Android)** - 移动端专用，成功率92%
3. **桌面 Safari (macOS)** - 苹果用户，成功率93%

### 模板选择建议
- **普通用户**: 使用"桌面 Chrome (Windows)"
- **移动操作**: 使用"移动 Chrome (Android)"
- **苹果设备**: 使用"桌面 Safari (macOS)"
- **隐私需求**: 使用"桌面 Firefox (Windows)"

## 配置优化技巧

### 1. 设备一致性
- 确保设备类型、操作系统、浏览器版本匹配真实设备
- 避免不合理的组合（如iOS设备使用Chrome浏览器）

### 2. 指纹多样性
- 定期更换User Agent
- 调整屏幕分辨率
- 更新字体列表
- 修改时区和语言设置

### 3. 行为自然性
- 鼠标移动：启用曲线概率（0.3-0.5）
- 键盘输入：设置合理的错误率（0.01-0.05）
- 页面交互：启用悬停和点击随机性
- 滚动行为：启用方向变化

### 4. 测试验证
- 创建配置后立即测试
- 分析测试结果中的警告
- 根据测试结果调整参数
- 定期重新测试现有配置

## 故障排除

### 常见问题
| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 配置测试失败 | 网络问题 | 检查网络连接，重试测试 |
| 导入配置失败 | 文件格式错误 | 检查JSON格式，使用示例模板 |
| 批量操作卡住 | 数据量太大 | 减少批量数量，分批操作 |
| 界面显示异常 | 浏览器缓存 | 清除缓存，刷新页面 |

### 调试步骤
1. 检查浏览器控制台错误
2. 验证网络请求是否成功
3. 检查API响应数据格式
4. 查看操作日志和错误信息

## API参考

### 主要接口
```typescript

GET /anti-detection-configs

// 创建配置
POST /anti-detection-configs

// 更新配置
PATCH /anti-detection-configs/:id

// 删除配置
DELETE /anti-detection-configs/:id

// 测试配置
```
