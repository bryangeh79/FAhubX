# 系统架构设计

## 架构概述
Facebook Auto Bot采用微服务架构思想，但以单体应用起步，确保快速开发和部署。随着业务增长，可以逐步拆分为微服务。

## 多租户架构

### 数据隔离策略
采用 **数据库级隔离** 与 **应用级隔离** 结合：

1. **数据库模式**: 每个租户有独立的schema
2. **数据行级**: 所有表包含`tenant_id`字段
3. **应用中间件**: 自动注入tenant上下文

### 租户生命周期
```
创建租户 → 初始化资源 → 激活服务 → 监控使用 → 续费/升级 → 暂停/删除
```

## 数据流设计

### 发布流程
```
用户创建内容 → 内容审核 → 调度设置 → 队列等待 → 
Facebook API调用 → 发布成功 → 效果追踪 → 报告生成
```

### 消息回复流程
```
Facebook Webhook → 消息接收 → 意图识别 → 
回复策略选择 → 生成回复 → 发送回复 → 对话记录
```

### 数据分析流程
```
原始数据收集 → 数据清洗 → 指标计算 → 
聚合分析 → 报告生成 → 可视化展示
```

## 技术决策

### 为什么选择NestJS？
- TypeScript原生支持
- 模块化架构
- 依赖注入
- 丰富的生态系统
- 企业级特性

### 为什么选择PostgreSQL？
- JSONB支持 (灵活的数据结构)
- 强大的查询能力
- 事务完整性
- 扩展性好
- 社区活跃

### 为什么选择React？
- 组件化开发
- 丰富的生态系统
- 性能优秀
- 学习曲线平缓
- 社区支持好
