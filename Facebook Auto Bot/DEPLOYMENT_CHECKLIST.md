# Facebook Auto Bot 生产环境部署检查清单

## 1. 部署前检查

### 1.1 系统要求检查
- [ ] 服务器操作系统: Ubuntu 20.04 LTS 或更高版本
- [ ] CPU: 至少 2 核
- [ ] 内存: 至少 4GB
- [ ] 磁盘空间: 至少 20GB 可用空间
- [ ] Docker: 版本 20.10 或更高
- [ ] Docker Compose: 版本 2.0 或更高

### 1.2 网络配置检查
- [ ] 防火墙已配置（开放端口: 80, 443, 22）
- [ ] 域名解析已配置（A记录指向服务器IP）
- [ ] SSL证书已准备（或使用Let's Encrypt）
- [ ] 服务器时间已同步（NTP）

### 1.3 安全配置检查
- [ ] SSH密钥认证已配置
- [ ] 禁用root SSH登录
- [ ] 防火墙规则已配置
- [ ] 系统更新已安装
- [ ] 非root用户已创建用于部署

## 紧急恢复流程

### 服务不可用
1. 检查健康状态: `./deployment-scripts/health-check.sh comprehensive`
2. 查看服务日志: `docker-compose -f docker-compose.prod.yml logs -f`
3. 重启问题服务: `docker-compose -f docker-compose.prod.yml restart [service]`
4. 如果问题持续，回滚到上一个版本

### 数据丢失
1. 停止写入操作
2. 从最新备份恢复: `./deployment-scripts/backup-restore.sh restore [backup-file]`
3. 验证数据完整性
4. 重新启动服务

### 安全事件
1. 立即隔离受影响系统
2. 收集证据和日志
3. 联系安全团队
4. 执行安全修复
5. 恢复服务运行
