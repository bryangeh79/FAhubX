## 系统概述

### 什么是 Facebook Auto Bot？
Facebook Auto Bot 是一个专业的 Facebook 自动化管理平台，提供以下核心功能：

- **账号管理**: 安全地管理多个 Facebook 账号
- **自动化任务**: 自动发帖、点赞、评论、消息回复
- **对话引擎**: 智能对话和客服自动化
- **数据分析**: 详细的执行报告和效果分析
- **团队协作**: 多用户协作和权限管理

### 系统架构
```
用户界面 (Web) → API 服务 → 任务引擎 → Facebook 平台
       ↓              ↓           ↓
   数据库 ←──── 监控系统 ←── 日志系统
```

### 适用场景
- **社交媒体营销**: 自动发布内容，提高曝光
- **客户服务**: 自动回复客户咨询
- **社群管理**: 管理 Facebook 群组和页面
- **数据分析**: 收集和分析社交媒体数据

## 环境准备

### 1.1 硬件要求
| 组件 | 最低要求 | 推荐配置 |
|------|---------|----------|
| CPU | 2核 | 4核 |
| 内存 | 4GB | 8GB |
| 存储 | 10GB | 50GB |
| 网络 | 10Mbps | 100Mbps |

### 1.2 软件要求
#### 自托管版本
- **操作系统**: Ubuntu 20.04+/CentOS 8+/Windows Server 2019+
- **Node.js**: v18.0.0 或更高版本
- **数据库**: PostgreSQL 14+ 和 Redis 6+
- **Docker**: 可选，简化部署

#### SaaS 版本（云端）
- 无需安装，直接访问 https://app.fbautobot.com
- 支持所有现代浏览器（Chrome 90+, Firefox 88+, Safari 14+）

### 1.3 网络要求
- **出站连接**: 需要访问 Facebook API (graph.facebook.com)
- **入站连接**: 如果需要 Webhook，需要公网 IP
- **代理/VPN**: 支持 HTTP/HTTPS/SOCKS5 代理

### 1.4 账号准备
1. **Facebook 开发者账号**（可选）:
   - 用于创建应用和获取 API 权限
   - 访问 https://developers.facebook.com

2. **Facebook 业务账号**（推荐）:
   - 商业账号更稳定，功能更完整
   - 建议使用专门的管理账号

## 系统安装

### 2.1 SaaS 版本（推荐新手）
1. **注册账号**
   - 访问 https://app.fbautobot.com
   - 点击"免费试用"或"立即注册"
   - 填写邮箱、密码、公司信息

2. **验证邮箱**
   - 检查注册邮箱，点击验证链接
   - 验证后自动登录系统

3. **完成初始设置**
   - 选择套餐（免费版/专业版/企业版）
   - 设置团队名称和时区
   - 邀请团队成员（可选）

### 2.2 自托管版本（Docker 部署）
#### 步骤 1: 下载安装包
```bash
# 创建项目目录
mkdir facebook-auto-bot
cd facebook-auto-bot

# 下载 docker-compose 文件
curl -O https://raw.githubusercontent.com/fbautobot/docker/main/docker-compose.yml
curl -O https://raw.githubusercontent.com/fbautobot/docker/main/.env.example
```

#### 步骤 2: 配置环境变量
```bash
# 复制环境变量模板
cp .env.example .env

# 编辑环境变量
nano .env
```

需要配置的关键变量：
```env
# 数据库配置
POSTGRES_DB=fbautobot
POSTGRES_USER=fbautobot_user
POSTGRES_PASSWORD=your_secure_password

# Redis 配置
REDIS_PASSWORD=your_redis_password

# 应用配置
APP_SECRET=your_app_secret_key
API_URL=http://localhost:3000
FRONTEND_URL=http://localhost:8080

# 邮件配置（可选）
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASSWORD=your_app_password
```

#### 步骤 3: 启动服务
```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

#### 步骤 4: 访问系统
- 前端界面: http://localhost:8080
- API 服务: http://localhost:3000
- 默认管理员账号: admin@example.com / admin123

### 2.3 自托管版本（手动安装）
详细手动安装指南请参考 [高级安装指南](../advanced/INSTALLATION_ADVANCED.md)。

## 账号配置

### 3.1 登录系统
1. 访问系统地址（SaaS 或自托管）
2. 输入用户名和密码
3. 如果是首次登录，需要修改默认密码

### 3.2 系统初始化
#### 3.2.1 个人信息设置
1. 点击右上角用户头像 → "个人设置"
2. 完善个人信息：
   - 姓名、职位、联系方式
   - 时区、语言偏好
   - 通知设置（邮件、站内信）

#### 3.2.2 团队设置（可选）
1. 进入"团队管理" → "创建团队"
2. 设置团队信息：
   - 团队名称、描述
   - 时区、工作日设置
3. 邀请团队成员：
   - 输入邮箱地址
   - 分配角色（管理员、编辑、查看者）

### 3.3 添加 Facebook 账号
#### 3.3.1 单个账号添加
1. 进入"账号管理" → "添加账号"
2. 填写账号信息：
   ```
   账号类型: [个人账号/商业账号/测试账号]
   邮箱/手机号: your_facebook_email@example.com
   密码: ********
   昵称: 我的主账号（用于识别）
   标签: [营销、客服、测试]（可选）
   ```
3. 点击"测试连接"验证账号
4. 保存账号

#### 3.3.2 批量导入账号
1. 准备 CSV 文件格式：
   ```csv
   email,password,nickname,tags,type
   account1@example.com,pass123,账号1,营销,商业
   account2@example.com,pass456,账号2,客服,个人
   ```
2. 进入"账号管理" → "批量导入"
3. 上传 CSV 文件
4. 预览并确认导入

#### 3.3.3 账号安全建议
1. **使用专用账号**: 不要使用个人主账号
2. **启用二次验证**: 在 Facebook 设置中启用
3. **定期更换密码**: 建议每 3 个月更换
4. **监控账号状态**: 系统会监控账号异常

### 3.4 账号分组管理
1. **创建分组**:
   - 进入"账号管理" → "分组"
   - 点击"新建分组"
   - 设置分组名称和描述

2. **分配账号到分组**:
   - 在账号列表勾选账号
   - 点击"分配到分组"
   - 选择目标分组

3. **分组用途示例**:
   - **营销组**: 用于内容发布和推广
   - **客服组**: 用于客户服务和回复
   - **测试组**: 用于测试新功能

## 第一个任务

### 4.1 了解任务类型
系统支持以下任务类型：

| 任务类型 | 描述 | 适用场景 |
|---------|------|----------|
| 发布帖子 | 在个人主页或页面发帖 | 内容营销、产品发布 |
| 点赞帖子 | 自动点赞指定帖子 | 互动提升、社群活跃 |
| 发表评论 | 在帖子下发表评论 | 用户互动、客户服务 |
| 发送消息 | 发送私信给用户 | 客户沟通、销售跟进 |
| 回复消息 | 自动回复收到的消息 | 客服自动化 |
| 加入群组 | 自动加入 Facebook 群组 | 社群拓展 |
| 分享内容 | 分享帖子到其他位置 | 内容分发 |

### 4.2 创建第一个发帖任务
#### 步骤 1: 进入任务创建页面
1. 点击左侧菜单"任务调度"
2. 点击"创建新任务"按钮
3. 选择任务类型"发布帖子"

#### 步骤 2: 配置任务基本信息
```yaml
任务名称: 我的第一个自动发帖
任务描述: 每天自动发布一条问候帖
优先级: 普通
执行账号: [选择刚才添加的账号]
```

#### 步骤 3: 配置帖子内容
```yaml
发布位置: 个人主页
帖子内容: 大家好！这是通过 Facebook Auto Bot 自动发布的帖子。😊
         每天问候，保持活跃！
         
         #自动化 #社交媒体 #科技
附加图片: [可选上传图片]
链接: [可选添加链接]
```

#### 步骤 4: 配置执行计划
```yaml
执行方式: 定时执行
执行时间: 每天 09:00
时区: 亚洲/上海 (GMT+8)
开始日期: 今天
结束日期: 无限制
```

#### 步骤 5: 高级设置（可选）
```yaml
失败重试: 启用，最多重试 3 次
执行延迟: 随机延迟 1-5 分钟（避免检测）
内容变量: 使用 {date} 自动插入日期
监控告警: 任务失败时发送邮件通知
```

#### 步骤 6: 保存并测试
1. 点击"保存"按钮
2. 点击"立即测试"验证任务
3. 查看测试结果，确认帖子已发布

### 4.3 任务状态说明
| 状态 | 图标 | 说明 |
|------|------|------|
| 等待中 | ⏳ | 任务已创建，等待执行时间 |
| 执行中 | 🔄 | 任务正在执行 |
| 成功 | ✅ | 任务执行成功 |
| 失败 | ❌ | 任务执行失败 |
| 暂停 | ⏸️ | 任务被手动暂停 |
| 已取消 | 🚫 | 任务被取消 |

### 4.4 查看任务结果
1. **任务列表**: 查看所有任务状态
2. **执行历史**: 查看每次执行的详细记录
3. **结果详情**: 点击任务查看执行详情和截图
4. **统计报表**: 查看任务成功率、执行时间等统计

## 系统监控

### 5.1 仪表板概览
登录后首页显示关键指标：
- **系统状态**: 服务运行状态
- **账号状态**: 在线/异常账号数量
- **任务统计**: 今日成功/失败任务数
- **性能指标**: 系统响应时间、资源使用

### 5.2 实时监控
1. **当前任务**: 查看正在执行的任务
2. **账号活动**: 查看账号的实时活动
3. **系统日志**: 查看实时系统日志
4. **性能监控**: CPU、内存、网络使用情况

### 5.3 告警设置
#### 5.3.1 告警类型
- **账号告警**: 账号异常、登录失败
- **任务告警**: 任务连续失败、执行超时
- **系统告警**: 资源不足、服务异常
- **安全告警**: 异常登录、可疑操作

#### 5.3.2 设置告警规则
1. 进入"系统设置" → "告警管理"
2. 点击"新建告警规则"
3. 配置告警条件：
   ```yaml
   告警名称: 账号登录失败告警
   触发条件: 账号连续登录失败 3 次
   检查频率: 每 5 分钟
   告警级别: 高
   ```
4. 配置通知方式：
   - 站内消息
   - 邮件通知
   - 短信通知（企业版）
   - Webhook（集成到其他系统）

### 5.4 数据报表
#### 5.4.1 日报表
- 每日自动生成，早上 8 点发送
- 包含前一天的执行统计
- 关键指标趋势分析

#### 5.4.2 自定义报表
1. 进入"数据分析" → "报表中心"
2. 点击"新建报表"
3. 选择数据维度：
   - 时间范围（今天、本周、本月、自定义）
   - 账号筛选（全部、分组、单个）
   - 任务类型筛选
4. 选择展示图表：
   - 柱状图、折线图、饼图
   - 数据表格、KPI 卡片

#### 5.4.3 报表导出
支持导出格式：
- **CSV**: 用于 Excel 分析
- **Excel**: 包含格式和图表
- **PDF**: 用于打印和分享
- **JSON**: 用于程序处理

## 常见问题

### 6.1 安装问题
#### Q: Docker 启动失败怎么办？
**A**: 常见原因和解决方案：
1. **端口冲突**: 修改 docker-compose.yml 中的端口
2. **权限不足**: 使用 sudo 或调整目录权限
3. **内存不足**: 增加 Docker 内存限制
4. **网络问题**: 检查网络连接和代理设置

#### Q: 数据库连接失败？
**A**: 检查步骤：
1. 确认 PostgreSQL 服务正在运行
2. 检查 .env 文件中的数据库配置
3. 确认防火墙允许数据库端口（默认 5432）
4. 查看日志获取详细错误信息

### 6.2 账号问题
#### Q: 账号测试连接失败？
**A**: 可能原因：
1. **账号密码错误**: 重新输入确认
2. **两步验证**: 需要输入验证码
3. **账号异常**: 账号可能被限制
4. **网络问题**: 检查网络连接和代理

**解决方案**:
1. 在浏览器中手动登录确认账号正常
2. 暂时禁用两步验证
3. 使用 Facebook 账号检查工具验证状态

#### Q: 如何避免账号被封？
**A**: 安全建议：
1. **合理频率**: 不要设置过高的执行频率
2. **内容合规**: 发布内容符合 Facebook 政策
3. **行为模拟**: 启用"人类行为模拟"功能
4. **多账号轮换**: 使用多个账号轮换执行
5. **定期休息**: 设置任务执行间隔和休息时间

### 6.3 任务问题
#### Q: 任务执行失败但没有错误信息？
**A**: 排查步骤：
1. 查看任务执行详情和截图
2. 检查系统日志中的详细错误
3. 手动执行相同操作验证
4. 联系技术支持提供日志分析

#### Q: 如何提高任务成功率？
**A**: 优化建议：
1. **增加延迟**: 在任务间增加随机延迟
2. **使用代理**: 为不同账号使用不同 IP
3. **内容多样化**: 避免发布重复内容
4. **时间优化**: 在用户活跃时段执行
5. **监控调整**: 根据执行结果调整策略

### 6.4 性能问题
#### Q: 系统运行缓慢怎么办？
**A**: 优化建议：
1. **硬件升级**: 增加内存和 CPU
2. **数据库优化**: 添加索引，清理旧数据
3. **缓存启用**: 确保 Redis 缓存正常工作
4. **任务优化**: 减少并发任务数量
5. **网络优化**: 优化网络连接和代理
