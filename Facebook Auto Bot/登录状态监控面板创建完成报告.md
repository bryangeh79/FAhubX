## 后端API需求（待实现）

为了完全功能，后端需要提供以下API：

### 核心API
1. `GET /api/v1/login-status` - 获取登录状态列表
2. `GET /api/v1/login-status/:id` - 获取单个账号状态
3. `GET /api/v1/login-status/stats` - 获取统计数据
4. `POST /api/v1/login-status/refresh` - 手动刷新状态

### 操作API
1. `POST /api/v1/login-status/:id/retry` - 重试登录
2. `DELETE /api/v1/login-status/:id/session` - 清除会话
3. `POST /api/v1/login-status/batch/retry` - 批量重试
4. `POST /api/v1/login-status/batch/clear-sessions` - 批量清除会话

### 配置API
1. `GET /api/v1/login-status/reconnect-config` - 获取重连配置
2. `PATCH /api/v1/login-status/reconnect-config` - 更新重连配置
3. `GET /api/v1/login-status/alerts` - 获取告警
4. `PUT /api/v1/login-status/alerts/:id/acknowledge` - 确认告警

### 数据API
1. `GET /api/v1/login-status/trends` - 获取趋势数据
2. `GET /api/v1/login-status/export` - 导出报告

## 部署和使用说明

### 前端部署
1. 确保所有依赖已安装：`npm install`
2. 构建项目：`npm run build`
3. 部署到Web服务器

### 使用流程
1. 用户登录系统
2. 点击侧边栏"登录状态监控"菜单
3. 查看所有账号的实时状态
4. 使用过滤功能查找特定账号
5. 点击"查看详情"查看详细信息
6. 配置自动重连规则（可选）
7. 监控健康状态和处理告警

### 配置说明
1. **自动刷新**：可在设置中配置刷新间隔
2. **重连策略**：根据账号重要性选择不同策略
3. **通知设置**：配置状态变化通知方式
4. **过滤规则**：保存常用的过滤条件
