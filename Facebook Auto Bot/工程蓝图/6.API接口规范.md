# 6. API接口规范

## API设计原则

### RESTful原则
1. **资源导向**：API围绕资源设计
2. **HTTP方法**：正确使用GET、POST、PUT、DELETE
3. **状态码**：使用标准HTTP状态码
4. **版本控制**：API版本在URL中体现

### 安全性原则
1. **认证**：所有API需要认证（除公开API）
2. **授权**：基于角色的访问控制
3. **限流**：防止API滥用
4. **审计**：记录所有API调用

### 性能原则
1. **分页**：列表接口必须支持分页
2. **过滤**：支持字段过滤和条件查询
3. **排序**：支持多字段排序
4. **缓存**：适当使用缓存头

## API基础信息

### 基础URL
```
生产环境: https://api.fbautobot.com/v1
开发环境: https://api-dev.fbautobot.com/v1
本地环境: http://localhost:3000/v1
```

### 认证方式
```yaml
认证头:
  Authorization: Bearer {access_token}

令牌类型:
  - Access Token: 短期令牌（1小时）
  - Refresh Token: 长期令牌（7天）

刷新令牌:
  POST /auth/refresh
  Body: { "refresh_token": "string" }
```

### 公共响应格式
```typescript
interface ApiResponse<T> {
  success: boolean;
  data?: T;
  error?: {
    code: string;
    message: string;
    details?: any;
  };
  meta?: {
    page?: number;
    limit?: number;
    total?: number;
    hasMore?: boolean;
  };
  timestamp: string;
}
```

### 错误码规范
```yaml
HTTP状态码:
  200: 成功
  201: 创建成功
  400: 请求错误
  401: 未认证
  403: 无权限
  404: 资源不存在
  429: 请求过多
  500: 服务器错误

业务错误码:
  - AUTH_*: 认证相关错误
  - USER_*: 用户相关错误
  - ACCOUNT_*: 账号相关错误
  - TASK_*: 任务相关错误
  - SYSTEM_*: 系统相关错误
```
