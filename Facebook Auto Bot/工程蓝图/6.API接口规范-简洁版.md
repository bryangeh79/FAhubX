# 6. API接口规范（简洁版）

## API概览

### 基础信息
- **基础URL**: `https://api.fbautobot.com/v1`
- **认证**: Bearer Token (JWT)
- **响应格式**: JSON
- **版本**: v1

### 通用响应格式
```json
{
  "success": boolean,
  "data": any,
  "error": {
    "code": string,
    "message": string
  },
  "meta": {
    "page": number,
    "limit": number,
    "total": number
  }
}
```

## 核心API端点

### 1. 认证API
```http
POST /auth/register     # 用户注册
POST /auth/login        # 用户登录
POST /auth/refresh      # 刷新令牌
GET  /auth/me           # 获取当前用户
POST /auth/logout       # 用户登出
```

### 2. 用户API
```http
GET    /users/{id}                    # 获取用户信息
PUT    /users/{id}                    # 更新用户资料
PUT    /users/{id}/preferences        # 更新用户偏好
GET    /users/{id}/stats              # 获取用户统计
```

### 3. 账号管理API
```http
GET    /accounts                      # 获取账号列表
POST   /accounts                      # 创建新账号
GET    /accounts/{id}                 # 获取账号详情
PUT    /accounts/{id}                 # 更新账号信息
DELETE /accounts/{id}                 # 删除账号
PUT    /accounts/{id}/vpn             # 更新VPN配置
POST   /accounts/{id}/test-connection # 测试连接
POST   /accounts/{id}/login           # 手动登录
GET    /accounts/{id}/health          # 账号健康检查
```

### 4. 任务管理API
```http
GET    /tasks/configs                 # 获取任务配置列表
POST   /tasks/conversation            # 创建对话任务
POST   /tasks/post                    # 创建发帖任务
GET    /tasks/{id}                    # 获取任务详情
PUT    /tasks/{id}                    # 更新任务配置
DELETE /tasks/{id}                    # 删除任务
PUT    /tasks/{id}/status             # 更新任务状态
POST   /tasks/{id}/execute            # 立即执行任务
GET    /tasks/{id}/history            # 获取执行历史
```

### 5. 对话剧本API
```http
GET    /conversation/scripts          # 获取剧本列表
GET    /conversation/scripts/{id}     # 获取剧本详情
POST   /conversation/scripts          # 创建剧本（管理员）
PUT    /conversation/scripts/{id}     # 更新剧本（管理员）
DELETE /conversation/scripts/{id}     # 删除剧本（管理员）
GET    /conversation/history          # 获取对话历史
```

### 6. 执行监控API
```http
GET    /executions                    # 获取执行记录
GET    /executions/{id}               # 获取执行详情
GET    /executions/{id}/logs          # 获取执行日志
POST   /executions/{id}/cancel        # 取消执行
GET    /monitoring/queue              # 获取队列状态
GET    /monitoring/system             # 获取系统状态
```

### 7. 通知API
```http
GET    /notifications                 # 获取通知列表
GET    /notifications/unread          # 获取未读通知
PUT    /notifications/{id}/read       # 标记为已读
PUT    /notifications/read-all        # 标记所有为已读
DELETE /notifications/{id}            # 删除通知
GET    /notifications/preferences     # 获取通知偏好
PUT    /notifications/preferences     # 更新通知偏好
```

### 8. 失败记录API
```http
GET    /failures                      # 获取失败记录
GET    /failures/{id}                 # 获取失败详情
PUT    /failures/{id}/status          # 更新失败状态
POST   /failures/{id}/retry           # 重试失败任务
GET    /failures/stats                # 失败统计
```

## 关键API详细说明

### 创建对话任务
```http
POST /tasks/conversation
Content-Type: application/json
Authorization: Bearer {token}

请求体:
{
  "name": "账号1和2日常聊天",
  "script_id": "script_001",
  "initiator_account_id": "account_001",
  "responder_account_id": "account_002",
  "schedule": {
    "type": "recurring",
    "frequency": "weekly",
    "days_of_week": [1, 3, 5],
    "time_of_day": "09:00",
    "timezone": "Asia/Kuala_Lumpur"
  },
  "personalization": {
    "use_account_names": true
  }
}

响应:
{
  "success": true,
  "data": {
    "id": "task_123",
    "name": "账号1和2日常聊天",
    "next_execution": "2026-04-15T09:00:00+08:00",
    "status": "active"
  }
}
```

### 立即执行任务
```http
POST /tasks/{taskId}/execute
Authorization: Bearer {token}

响应:
{
  "success": true,
  "data": {
    "execution_id": "exec_456",
    "status": "pending",
    "queue_position": 0,
    "estimated_start": "2026-04-12T09:05:00Z"
  }
}
```

### 获取执行状态
```http
GET /executions/{executionId}
Authorization: Bearer {token}

响应:
{
  "success": true,
  "data": {
    "id": "exec_456",
    "task_id": "task_123",
    "status": "running",
    "progress": 2,
    "total_steps": 6,
    "current_step": "正在发送第二条消息",
    "start_time": "2026-04-12T09:05:00Z",
    "estimated_completion": "2026-04-12T09:12:00Z"
  }
}
```

## 查询参数规范

### 通用查询参数
```yaml
分页:
  - page: 页码 (默认: 1)
  - limit: 每页数量 (默认: 20, 最大: 100)

排序:
  - sort: 排序字段
  - order: asc|desc (默认: desc)

过滤:
  - status: 状态过滤
  - created_after: 创建时间之后
  - created_before: 创建时间之前
```

### 账号查询参数
```yaml
GET /accounts
参数:
  - status: active|disabled|banned|suspended
  - tags: 标签过滤（逗号分隔）
  - vpn_provider: openvpn|wireguard|nordvpn
  - has_session: true|false
```

### 任务查询参数
```yaml
GET /tasks/configs
参数:
  - type: conversation|post|like|share
  - status: active|paused|completed|error
  - account_id: 关联账号ID
  - next_execution_after: 下次执行时间之后
```

### 执行记录查询参数
```yaml
GET /executions
参数:
  - task_id: 任务ID
  - account_id: 账号ID
  - status: completed|failed|cancelled
  - start_date: 开始日期
  - end_date: 结束日期
```

## 错误处理

### HTTP状态码
```yaml
200: 成功
201: 创建成功
400: 请求参数错误
401: 未认证
403: 无权限
404: 资源不存在
409: 资源冲突（如账号已被锁定）
429: 请求过于频繁
500: 服务器内部错误
```

### 业务错误码
```yaml
AUTH_001: 无效的认证令牌
AUTH_002: 令牌已过期
AUTH_003: 权限不足

USER_001: 用户不存在
USER_002: 邮箱已被注册
USER_003: 用户名已被使用

ACCOUNT_001: 账号不存在
ACCOUNT_002: 账号已被锁定
ACCOUNT_003: VPN配置错误
ACCOUNT_004: 登录失败

TASK_001: 任务不存在
TASK_002: 任务已被暂停
TASK_003: 并发限制（账号正忙）
TASK_004: 调度时间冲突

SYSTEM_001: 服务暂时不可用
SYSTEM_002: 数据库连接错误
SYSTEM_003: 外部服务错误
```

## 限流策略

### API限流规则
```yaml
认证相关:
  - /auth/*: 10次/分钟/IP

用户相关:
  - /users/*: 60次/分钟/用户

账号相关:
  - /accounts/*: 30次/分钟/用户

任务相关:
  - /tasks/*: 20次/分钟/用户

执行相关:
  - /executions/*: 60次/分钟/用户
```

### 并发限制
```yaml
账号级别:
  - 一个账号同时只能执行一个任务

用户级别:
  - 一个用户最多10个并发任务

系统级别:
  - 全局最多1000个并发任务
```

## WebSocket API

### 连接建立
```javascript
const ws = new WebSocket('wss://api.fbautobot.com/v1/ws');

// 认证消息
ws.send(JSON.stringify({
  type: 'auth',
  token: 'bearer_token'
}));
```

### 消息类型
```yaml
订阅:
  - type: 'subscribe', channel: 'task_updates'
  - type: 'subscribe', channel: 'notifications'
  - type: 'subscribe', channel: 'system_alerts'

取消订阅:
  - type: 'unsubscribe', channel: 'channel_name'
```

### 推送消息
```json
{
  "type": "task_update",
  "channel": "task_updates",
  "data": {
    "task_id": "task_123",
    "execution_id": "exec_456",
    "status": "running",
    "progress": 50,
    "timestamp": "2026-04-12T09:10:00Z"
  }
}
```

## 数据格式

### 时间格式
- 所有时间使用ISO 8601格式
- 时区: UTC或带时区的ISO格式
- 示例: `2026-04-12T09:00:00Z` 或 `2026-04-12T17:00:00+08:00`

### 分页格式
```json
{
  "meta": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "total_pages": 8,
    "has_previous": false,
    "has_next": true
  }
}
```

### 过滤格式
```json
{
  "filters": {
    "status": ["active", "paused"],
    "created_after": "2026-04-01T00:00:00Z",
    "tags": ["primary", "business"]
  }
}
```

## 版本管理

### 版本策略
- 版本在URL中体现: `/v1/`
- 向后兼容的修改不升级主版本
- 破坏性变更需要升级主版本
- 旧版本API保持可用至少6个月

### 弃用流程
1. 在文档中标记为"已弃用"
2. 返回`Deprecation`头
3. 6个月后移除
4. 提供迁移指南
