# 4. 核心模块设计

## 模块化架构概述

### 设计原则
1. **单一职责**：每个模块只负责一个明确的功能
2. **松耦合**：模块间通过明确定义的接口通信
3. **高内聚**：相关功能组织在同一个模块中
4. **可测试**：模块独立可测试，依赖可模拟
5. **可替换**：模块实现可替换，不影响系统其他部分

## 模块详细设计

### 1. 用户模块 (User Module)

#### 功能职责
- 用户注册、登录、认证
- 用户资料管理
- 权限和角色管理
- 会话管理

#### 核心接口
```typescript
interface UserService {
  // 用户管理
  register(userData: RegisterDto): Promise<User>;
  login(credentials: LoginDto): Promise<AuthToken>;
  logout(userId: string): Promise<void>;
  updateProfile(userId: string, profile: UpdateProfileDto): Promise<User>;
  
  // 权限管理
  assignRole(userId: string, role: UserRole): Promise<void>;
  checkPermission(userId: string, permission: string): Promise<boolean>;
  
  // 会话管理
  createSession(userId: string, deviceInfo: DeviceInfo): Promise<Session>;
  validateSession(sessionId: string): Promise<SessionValidationResult>;
  revokeSession(sessionId: string): Promise<void>;
}
```

#### 数据模型
```typescript
interface User {
  id: string;
  email: string;
  username: string;
  passwordHash: string;
  status: 'active' | 'suspended' | 'deleted';
  roles: UserRole[];
  preferences: UserPreferences;
  createdAt: Date;
  updatedAt: Date;
}

interface UserPreferences {
  notificationSettings: NotificationSettings;
  uiPreferences: UIPreferences;
  privacySettings: PrivacySettings;
}

interface Session {
  id: string;
  userId: string;
  deviceInfo: DeviceInfo;
  accessToken: string;
  refreshToken: string;
  expiresAt: Date;
  createdAt: Date;
}
```

### 5. 调度模块 (Scheduler Module)

#### 功能职责
- 秒级精度任务调度
- 并发控制和资源分配
- 队列管理和优先级处理
- 失败重试和恢复

#### 核心接口
```typescript
interface SchedulerService {
  // 调度管理
  scheduleTask(task: ScheduledTask): Promise<string>;
  rescheduleTask(taskId: string, newSchedule: Schedule): Promise<void>;
  cancelScheduledTask(taskId: string): Promise<void>;
  listScheduledTasks(filters?: ScheduleFilters): Promise<ScheduledTask[]>;
  
  // 队列管理
  getQueueStatus(queueName: string): Promise<QueueStatus>;
  pauseQueue(queueName: string): Promise<void>;
  resumeQueue(queueName: string): Promise<void>;
  clearQueue(queueName: string, status?: QueueClearOptions): Promise<number>;
  
  // 并发控制
  acquireLock(resourceId: string, ttl: number): Promise<boolean>;
  releaseLock(resourceId: string): Promise<void>;
  extendLock(resourceId: string, additionalTtl: number): Promise<boolean>;
  
  // 监控
  getSchedulerMetrics(timeRange: TimeRange): Promise<SchedulerMetrics>;
  getWorkerStatus(workerId?: string): Promise<WorkerStatus[]>;
  getFailedJobs(queueName: string, limit?: number): Promise<FailedJob[]>;
}
```

#### 数据模型
```typescript
interface ScheduledTask {

  timeout: number;

  resourceConstraints?: ResourceConstraint[];

  // 状态
  status: 'scheduled' | 'queued' | 'running' | 'completed' | 'failed';

  lastRun?: Date;

  errorCount: number;
  
  createdAt: Date;
  updatedAt: Date;

interface Schedule {

    delay?: number; // 延迟毫秒数
  };
  
  // Cron表达式
  cron?: {
    expression: string;
    timezone?: string;
  };
  
  // 间隔执行
  interval?: {
    milliseconds: number;
    startImmediately?: boolean;
  };
  
  // 指定时间
  date?: {
    timestamp: Date;
    timezone?: string;
  };
  
  // 重复配置
  repeat?: {
    count?: number;
    until?: Date;
  };
}

interface QueueStatus {
  name: string;
  waiting: number;
  active: number;
  completed: number;
  failed: number;
  delayed: number;
  paused: boolean;
  
  // 性能指标
  throughput: number; // 任务/分钟
  averageProcessingTime: number;
  errorRate: number;
  
  // 资源使用
  memoryUsage: number;
  cpuUsage: number;
  
  updatedAt: Date;
}

interface ResourceConstraint {
  type: 'account' | 'ip' | 'browser' | 'vpn';
  resourceId: string;
  exclusive: boolean; // 是否独占
  maxConcurrent?: number;
}
```
